UX Principles

These rules govern every design decision. Any component that violates these principles gets redesigned.

① Zero to output in under 3 seconds

A user lands on any tool page → pastes input → sees result. No configuration, no options to set, no reading required. Default settings must always produce useful output.

② Every label explains itself

No icon-only buttons on primary actions. Every input has a placeholder that says exactly what to paste. Every button label says exactly what will happen ("Format JSON", not just "Submit").

③ Errors help, not frustrate

If a user pastes invalid JSON, show "⚠ Invalid JSON — check line 3" not a generic red border. Error messages must always tell the user what went wrong AND what to do next.

④ One primary action per view

Each tool page has one clearly dominant action button. All other buttons are visually secondary. The user should never be confused about what to click first.

⑤ Confirm every user action

"Copy" button turns green and shows "✓ Copied!" for 2 seconds. "Clear" asks for confirmation. "Format" shows "✓ Valid JSON · 4 keys" in status bar. Users must always know their action worked.

⑥ Mobile is not an afterthought

Workspace panels stack vertically on mobile. Buttons have 44px minimum tap targets. Search is sticky. The tool must be fully usable on a phone — not just technically responsive.

⑦ Privacy is shown, not just stated

Small "🔒 Processed in your browser — nothing is uploaded" note under every tool workspace. Users should feel safe pasting sensitive data like passwords, tokens, and API keys.

⑧ Consistency over creativity

Every tool page looks identical in structure. Users learn the layout once. Breadcrumbs always in the same position. Copy button always bottom-left. Related tools always below the workspace.
