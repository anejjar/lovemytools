# 🌐 Website Plan: Developer Tools (SuperDevPro Clone)
**Focus:** SEO-first, tool-per-page architecture  
**Goal:** Rank for high-intent developer tool search queries  
**Stack Recommendation:** Next.js 14 (App Router) + Tailwind CSS + MDX for FAQs

---

## 🏗️ Site Architecture

```
/                          → Home Page
/extension                 → Extension Landing Page
/tools/[slug]              → Individual Tool Pages (dynamic)
/blog                      → (future) SEO blog content
/sitemap.xml               → Auto-generated
/robots.txt                → Crawl rules
```

## 📄 Individual Tool Page Structure

Every tool page (`/tools/[slug]`) follows this exact template:

```
┌─────────────────────────────────────────┐
│  Breadcrumb: Home > Tools > [Tool Name] │
├─────────────────────────────────────────┤
│  HERO                                   │
│  H1: [Tool Name] — Free Online Tool     │
│  Short description (150–160 chars)      │
│  Trust badges: Fast · Private · Free   │
├─────────────────────────────────────────┤
│  WORKING TOOL (Interactive)             │
│  Input panel + Output panel             │
│  Action buttons: Copy, Clear, Download  │
├─────────────────────────────────────────┤
│  ABOUT THIS TOOL (SEO Text)             │
│  What it is, how it works, use cases   │
│  300–500 words, naturally keyword-rich  │
├─────────────────────────────────────────┤
│  EXTENSION CTA BANNER                   │
│  "Use [Tool Name] directly in Chrome"  │
│  [Install Free Extension] button        │
├─────────────────────────────────────────┤
│  HOW TO USE (Step-by-step)              │
│  Numbered steps with screenshots/GIFs  │
│  Good for featured snippets             │
├─────────────────────────────────────────┤
│  RELATED TOOLS (Card Grid)              │
│  4–6 tools from same/related category  │
│  Internal linking for SEO              │
├─────────────────────────────────────────┤
│  FAQS (Tool-Specific)                   │
│  5–7 questions with schema markup      │
│  Targets "People Also Ask" in Google   │
├─────────────────────────────────────────┤
│  FOOTER                                 │
└─────────────────────────────────────────┘
```

### SEO Metadata per Tool Page
```
Title:     [Tool Name] Online — Free & Fast | DevTools
Description: Use our free [Tool Name] online. [One benefit]. No signup needed. Works instantly in your browser.
OG Image:  Dynamic OG image with tool name + icon (use Vercel OG or Satori)
Schema:    WebApplication + FAQPage + BreadcrumbList
Canonical: https://yourdomain.com/tools/[slug]
```

## 🔍 SEO Strategy

### Keyword Strategy
- **Category pages** target broad terms: "json tools online", "css tools for developers"
- **Tool pages** target high-intent specific terms: "json formatter online free"
- **Extension page** targets: "chrome extension developer tools", "best chrome extensions for developers"

### Technical SEO Checklist
- [ ] Next.js `generateMetadata()` per tool page (dynamic titles/descriptions)
- [ ] Dynamic `sitemap.xml` covering all tool pages
- [ ] `robots.txt` allowing full crawl
- [ ] Structured data: `WebApplication`, `FAQPage`, `BreadcrumbList` on every page
- [ ] Dynamic Open Graph images (Vercel OG / Satori)
- [ ] Core Web Vitals: LCP < 2.5s, CLS < 0.1, FID < 100ms
- [ ] Canonical tags on all pages
- [ ] `hreflang` if multilingual in future
- [ ] Internal linking: every tool links to 4–6 related tools
- [ ] Breadcrumbs on all tool pages

### Content SEO
- [ ] Each tool page: 300–500 words of original descriptive content
- [ ] FAQ schema on every page (5–7 questions targeting PAA boxes)
- [ ] "How to use" section targeting featured snippets
- [ ] Alt text on all images/screenshots
- [ ] Tool name in URL, H1, first paragraph, and meta description

### Link Building (Future)
- Submit to: Product Hunt, Hacker News, dev.to, CSS-Tricks, Smashing Magazine directories
- GitHub repo with README linking to site
- Target "best online developer tools" roundup articles
